# 桌面中心鱼眼视觉标定任务

## 目标

在桌面几何中心固定视觉标记板，使用机械臂末端的鱼眼相机识别标记板，得到：

- 桌面中心在左机械臂基坐标系下的三维坐标；
- 桌面中心在右机械臂基坐标系下的三维坐标；
- 桌面的方向和法向；
- 将 UMI/Vive 轨迹绕桌面中心法向旋转 180° 所需的坐标变换。

本任务不使用 SLAM，采用“标记板位姿检测 + 手眼标定 + 机器人正运动学”。

## 坐标系约定

- `B_left`：左机械臂基坐标系；
- `B_right`：右机械臂基坐标系；
- `E`：机械臂末端法兰坐标系；
- `C`：鱼眼相机坐标系；
- `T`：桌面中心坐标系，也是标记板坐标系。

桌面坐标系建议定义为：

- 原点位于桌面几何中心；
- `+Z` 垂直桌面向上；
- `+X` 从 UMI 一侧指向 Franka 一侧；
- `+Y` 按右手定则确定。

## 要做的事情

### 1. 制作并安装标记板

- [ ] 选择 AprilGrid、ChArUco 或较大的 AprilTag；
- [ ] 打印后测量标记的真实尺寸，记录单位和尺寸定义；
- [ ] 测量桌面长、宽或两条对角线，确定桌面几何中心；
- [ ] 让标记板坐标原点与桌面中心重合；
- [ ] 让标记板平面与桌面平行，并记录板的朝向；
- [ ] 记录标记板厚度，必要时补偿其与桌面上表面的高度差；
- [ ] 固定标记板，防止测量期间移动或翘曲。

### 2. 标定鱼眼相机内参

- [ ] 固定相机分辨率、焦距和采集模式；
- [ ] 从不同距离、角度采集标定图像，覆盖画面中心和边缘；
- [ ] 使用适合鱼眼相机的模型完成内参标定；
- [ ] 保存焦距、主点、畸变参数、分辨率和相机模型名称；
- [ ] 检查全视场重投影误差，重点检查鱼眼图像边缘；
- [ ] 使用标定结果测试标记板角点检测和位姿估计。

### 3. 完成相机与机械臂的手眼标定

- [ ] 保持桌面标记板固定；
- [ ] 机械臂从至少 10～20 个不同位置和方向观察标记板；
- [ ] 每次同步记录机械臂末端位姿和标记板在相机下的位姿；
- [ ] 确保采样姿态包含足够的旋转变化，而不只是平移；
- [ ] 求出相机与机械臂末端之间的固定变换；
- [ ] 保存左、右机械臂各自的手眼标定结果；
- [ ] 用未参与标定的姿态验证手眼标定误差。

### 4. 测量桌面中心

- [ ] 左机械臂从 5～10 个不同姿态观察完整标记板；
- [ ] 每个姿态计算一次标记板中心在左机械臂基坐标系下的位置和方向；
- [ ] 对有效结果进行异常值剔除和平均，保存最终结果；
- [ ] 右机械臂重复相同测量；
- [ ] 检查不同观察姿态所得结果是否一致；
- [ ] 保存桌面坐标系相对左右机械臂基坐标系的完整变换，而不只保存中心点。

坐标变换链为：

```text
机械臂基座 -> 机械臂末端 -> 鱼眼相机 -> 标记板/桌面中心
```

### 5. 验证桌面坐标系

- [ ] 检查计算出的桌面 `+Z` 是否垂直桌面向上；
- [ ] 检查桌面 `+X` 是否指向 Franka 一侧；
- [ ] 在标记板中心上方设置一个安全测试点；
- [ ] 让机械臂低速移动到测试点，先在较高位置空跑；
- [ ] 使用尺具或探针检查实际位置误差；
- [ ] 确认米、毫米等单位没有混用；
- [ ] 确认旋转矩阵、四元数的方向和顺序约定一致。

### 6. 转换 UMI/Vive 轨迹

- [ ] 使用已有的 Vive 到机械臂基坐标系转换处理 UMI 轨迹；
- [ ] 将轨迹绕桌面中心的 `+Z` 轴旋转 180°；
- [ ] 同时旋转轨迹位置和末端姿态；
- [ ] 根据任务检查左右手轨迹是否需要交换；
- [ ] 保留原始轨迹，输出转换后的新轨迹文件；
- [ ] 离线绘制原轨迹、桌面中心和转换后轨迹进行检查；
- [ ] 在桌面上方低速空跑，确认无碰撞后再进行物体操作。

## 需要保存的标定文件

- 鱼眼相机内参和畸变参数；
- 标记板类型、编号、真实尺寸和坐标系定义；
- 左机械臂的手眼标定结果；
- 右机械臂的手眼标定结果；
- 桌面坐标系在左机械臂基坐标系下的位姿；
- 桌面坐标系在右机械臂基坐标系下的位姿；
- Vive 到机械臂基坐标系的变换；
- 标定日期、相机分辨率、机器人配置和误差统计。

## 验收标准

- 从不同机械臂姿态观察标记板时，桌面中心计算结果稳定；
- 左右机械臂对同一桌面测试点的测量结果相互一致；
- 转换后的轨迹确实位于桌面中心的对侧；
- 旋转后夹爪朝向、运动方向和左右手语义正确；
- 低速空跑无越界、奇异位形、自碰撞或桌面碰撞。

## 注意事项

- 不要把普通支付二维码当作高精度位姿标记；
- 不要使用普通针孔模型直接处理大视场鱼眼图像；
- 标记板中心必须与实际桌面中心对应，否则要记录二者的固定偏移；
- 标记尺寸填写错误会直接造成位置尺度错误；
- 相机图像与机器人末端位姿需要时间同步；
- 移动鱼眼相机、Vive 基站、机械臂底座或桌子后，应重新验证标定。
